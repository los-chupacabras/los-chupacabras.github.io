Tema: HTML for User Agents

        Cada diferente Browser de la red tiene su propio String que lo distingue del resto, esto es conocido como el USER AGENT(UA). El formato para distinguir al user agent es primero poner el nombre del programa seguido de la versión y luego se ponen detalles del browser/computadora. Los user agents se usan en momentos que el sitio de web no puede distinguir que tipo de computadora lo está accesando y le pide el user agent para poder saber. El punto de los user agents es poder tener un script de javascript que verifique el user agent y dejándose llevar por eso reduzca o agrande el tamaño de la página para acomodar al equipo que lo esté usando. Los user agents les hacen la vida mucho más fácil a los diseñadores de sitios de web. Es una gran ventaja porque lo único que se tiene que hacer es tener el script con el "if" y verifique su user agent.
        Los user agents se usan cuando un diseñador de páginas web desea tener páginas específicas para ciertos equipos electrónicos. Los user agents son efectivos y muy útiles a los diseñadores de páginas web, es necesario tener en mente que tipo de equipo electrónico podrá acusar su página. Es más una medida preventiva para que el usuario tenga una experiencia agradable en la página suya.
        Al poder ajustarse la página dependiendo del browser que lo acceda se puede limitar muchas cosas dependiendo del equipo que lo accede o simplemente se manda lo posible. Por ejemplo, si usas una versión vieja de Internet Explorer y entras a una página nueva que tiene objetos de HTML5, pues el User Agent le va decir al browser que no es compatible y no muestra el objeto. De la misma manera es posible hacer que algunos browsers se pongan un User Agent de otro browser para poder acceder la página con todas las funciones. Un ejemplo de esto sería el browser de los celulares Android que el User Agent que usa es el de Safari, el browser creado por Apple. Es posible también tener un browser que pueda cambiar su User Agent cuando el usuario lo escoja. Por más que es un beneficio a el diseñador usar User Agent strings, el W3C lo considera una mala práctica, ya que se a veces se necesita un browser específico para poder disfrutar de todos lo que puede tener una página web. La solución del W3C es usar HTML que sea estándar en todos los equipos/browsers posibles.

        Donde los User Agent strings verdaderamente demuestran su uso eficiente es con los equipos móviles y tabletas. Ayudan a que el momento que el usuario entre a la página, se decida si se usa el HTML original y si usar el HTML creado explícitamente para los móviles. Hay aquellos que piensan que es más eficiente hacer una página HTML que sea flexible con cualquier equipo o browser que lo vaya acceder, y por lo tanto por eso es que muchas páginas tienen diseños como para equipos con pantallas táctiles.
